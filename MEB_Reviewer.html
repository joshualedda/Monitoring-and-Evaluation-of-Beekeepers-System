Section 1 What is MEB?

📌 Full Name

MEB stands for Monitoring & Evaluation of Beekeepers. It is a software product and web application created for NARTDI (National Agriculture Research & Technology for Development Institute) to monitor and evaluate beekeeping activities across the Philippines.

🎯 Purpose

Groups information about beekeepers and colonies in one central place. Various actors (managers, admins) can easily follow all beekeepers, their activities, documents, production data, and inquiries.

🕒 Version

Version 1.0, released December 2019. The system supports both English and French languages.

🌐 Deployment

Can run in 3 modes:

  • Stand-alone (single PC)
  • Local Area Network (LAN)
  • Web / Internet (hosted)

Section 3 Concept of Operation

How the System Operates

  • A System Administrator (SA) must be designated first to oversee the entire system
  • SA installs the system and creates all user accounts
  • Each account includes: username, password, email, phone, gender, language, and access rights
  • Access rights = permissions for what each user can see and do
  • System settings must be configured before adding beekeepers and colonies
  • Follow the sequence in the manual — there is a hierarchy to the setup

Section 5 System Modules

🏠 Dashboard

The first screen after login. Shows an overview of beekeeping progress for the current year, total colonies, total active beekeepers, and latest news. Users select a year and click Submit to refresh the view. The dashboard also shows a map of the Philippines.

📰 News Module

Displays posts written by MEB users. Users with permission can Manage Posts. Clicking "View Post" opens the post and any attached documents. News categories are managed in Settings. Posts have a Date From/To and Active/Inactive status to control visibility.

All reports require parameters to be selected before generating. Reports are produced as PDF files. Adobe Acrobat Reader is recommended on the workstation.

⚠️ Important Note

The system will prompt you for required parameters before a report can be generated. Parameters vary between reports (e.g., province, year, bee type, date range).

Section 7 Settings Module

The Settings module defines all reference/lookup data used throughout the system. Only the System Administrator manages these settings. All settings follow the same Add/Edit/Delete pattern.

📍 Location Settings

  • Province
  • Municipality
  • City

👨‍🌾 Beekeeper Settings

  • Rating – scale for beekeeper evaluation
  • Association – group the beekeeper belongs to
  • Origin – origin of the beekeeper
  • Type of Support – support given after inquiry
  • Type of Inquiry – category of question/issue

🏡 Colony Settings

  • Bee Type – species/variety of bee
  • Status – current state of colony
  • Phase – current development stage

📰 News Settings

  • Category – news post categories
  • Post – manage, add, edit, delete news posts
  • Posts have Date From/To + Active/Inactive
  • Documents can be attached to posts

Section 8 User Management & Security

👤 User Accounts

Each user requires:

  • Username & Password
  • Email & Phone
  • Gender
  • Language preference (English/French)
  • Assigned Profile (access rights)

🔑 Profiles (Access Control)

A profile defines what a user can do. Permissions are granted per module:

  • Create – add new records
  • Update – edit existing records
  • View – read-only access
  • Delete – remove records

If no access is granted, the menu item is hidden from that user.

🙋 My Profile / My User

  • My Profile – users can view their own profile (read-only)
  • My User – users can update their own personal info (name, email, password, etc.)

🔐 Security Flow

  • Step 1: SA designates and installs the system
  • Step 2: SA creates user accounts with usernames & passwords
  • Step 3: SA assigns a Profile to each user (controls access)
  • Step 4: Users log in — only authorized modules appear in their menu
  • Step 5: Users interact with data based on their permissions

Section 9 Reportico & Documentation

📈 Reportico

A flexible web reporting tool built into MEB. Allows users to create, maintain, and run custom reports without programmer help. Requires knowledge of:

  • Database structure (schema)
  • MySQL query language

More info: www.reportico.org

📚 Documentation Module

Accessible from the main menu. Contains the official MEB User Guide in PDF format. Users can download or print the guide directly from within the system.
